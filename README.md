# Wordform Report Export
Это тестовое задание выполнил Волохов Прохор.

API принимает `.txt` файл, построчно анализирует его содержимое, приводит слова к нормальной форме и формирует `.xlsx`-отчет со статистикой:

- словоформа
- количество вхождений во всем документе
- количество вхождений в каждой строке

## Постановка задачи
Реализован endpoint:

`POST /public/report/export`

На вход передается текстовый файл.  
Файл может быть большим, поэтому обработка строится с учетом потокового чтения и без загрузки всего содержимого в память одной операцией.

Для каждой словоформы рассчитывается:

- общее количество вхождений в документе
- количество вхождений по строкам

Результат возвращается в формате `.xlsx`.

## Особенности реализации
- Слоистая архитектура
- доменная логика изолирована от конкретных библиотек
- нормализация словоформ через `pymorphy3`
- экспорт в Excel 
- построчная обработка файла
- Модульные и интеграционные тесты, также проверено вручную
- Упаковка в Docker-контейнер для удобства развертывания

## Оговорка
В текущей реализации файл обрабатывается построчно, а архитектура разделена на domain/application/infrastructure/api.
Для реальной эксплуатации сервиса с большим числом тяжёлых параллельных запросов следующим шагом было бы ограничение числа одновременно выполняемых обработок либо вынос генерации отчета в фоновый воркер.
